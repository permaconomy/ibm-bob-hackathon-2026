About This Project

Built for IBM Bob Dev Day Hackathon May 2026

What We Do

This is an AI-powered hackathon submission compliance checker that helps participants validate their entire submission against all 8 red flags before final submission. Our system prevents disqualification by catching compliance issues early.

Unlike generic code checkers, we specifically validate IBM Bob Dev Day Hackathon requirements: bob_sessions folder structure, exposed credentials, banned AI models, team roster limits, timestamp validation, video presence, deliverables completeness, and personal information scanning.

8 Red Flags We Validate

  • bob_sessions Folder

    Validates presence of required .md files and screenshots proving IBM Bob IDE usage

  • Exposed Credentials

    Scans for hardcoded API keys, passwords, and tokens in code

  • Banned AI Models

    Detects usage of restricted LLMs like llama-3-405b-instruct

  • Team Roster Validation

    Ensures maximum 5 members and no duplicates

  • Timestamp Verification

    Validates submission before May 3, 2026, 10:00 AM ET deadline

  • Video Presence

    Confirms required demo video file or embedded link

  • Deliverables Completeness

    Checks all 4 required components are present

  • Personal Information Scan

    Detects PI data and social media scrapes

Creative Innovation

Meta-Validation Architecture: Our solution is self-documenting and self-validating. Every component was built using IBM Bob IDE, and our own bob_sessions folder demonstrates perfect compliance with the rules we enforce.

RAG-Powered Intelligence: We use watsonx.ai embeddings with FAISS vector storage to create semantic understanding of compliance requirements. Teams can ask natural language questions and receive contextual answers.

Real-Time Feedback Loop: Unlike post-submission validation, we provide compliance reports during development, enabling iterative improvement and dramatically reducing disqualification risk.
